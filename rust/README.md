# EXE加密工具 (ExeLock)

一个给Windows EXE加密码保护的工具，加密后的EXE需要输入正确密码才能启动。

## 技术方案对比

| 方案 | 实现难度 | 兼容性 | 安全性 | 推荐度 |
|------|----------|--------|--------|--------|
| **封装器+内存加载(RunPE)** | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ✅ 推荐入门 |
| **PE节区注入加壳** | ⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ | 进阶 |
| 释放临时文件执行 | ⭐ | ⭐⭐⭐⭐⭐ | ⭐ | 不推荐 |

## 编程语言选择

| 语言 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **Rust** | 内存安全、无依赖、体积小、性能好、Windows API绑定完善 | 学习曲线稍陡 | ✅ 首选 |
| C++ | 最灵活、生态成熟、资料多 | 内存不安全、开发效率低 | 次选 |
| C# | 开发快 | 需要.NET运行时、体积大、易反编译 | 不推荐 |
| Python | 原型快 | 打包后体积极大、性能差 | 仅原型 |

## 项目结构

```
exe-lock/
├── Cargo.toml          # 工作空间配置
├── packer/             # 加密工具（命令行）
│   └── src/main.rs     # 加密逻辑：读取原EXE -> 加密 -> 嵌入Stub
└── stub/               # 运行时加载器（被嵌入的壳）
    └── src/main.rs     # 密码验证 -> 解密 -> 内存加载执行
```

## 核心原理

1. **加密阶段(Packer)**：
   - 读取目标EXE文件到内存
   - 使用用户密码派生密钥（PBKDF2）
   - AES-256-GCM加密整个EXE数据
   - 将加密数据作为二进制资源拼接到Stub EXE末尾
   - 生成最终的受保护EXE

2. **运行阶段(Stub)**：
   - 启动后弹出密码输入框
   - 使用输入的密码派生密钥
   - 解密自身附带的加密EXE数据
   - 在内存中手动加载PE（RunPE技术）
   - 处理重定位、导入表
   - 跳转到原始入口点(OEP)执行

## ✨ 特性

- ✅ **同目录释放**：解密后的EXE释放到原程序所在目录，完美解决DLL/配置文件/资源依赖问题
- ✅ **命令行参数透传**：加密后的程序可以正常接收命令行参数
- ✅ **正确工作目录**：启动时设置正确的工作目录，相对路径正常工作
- ✅ **自动清理**：程序退出后自动删除临时文件，下次启动也会清理遗留文件
- ✅ **权限降级**：当前目录不可写时自动fallback到临时目录
- ✅ AES-256-GCM 认证加密（防篡改）
- ✅ PBKDF2 密钥派生（防彩虹表）
- ✅ 可插拔加密算法接口，支持自定义算法
- ✅ 不修改原始EXE逻辑，兼容性好

## 使用方法

```bash
# 编译
cargo build --release

# 加密EXE
exe-lock pack --input target.exe --output locked.exe --password "yourpassword"

# 运行加密后的EXE时会弹出密码框
```
